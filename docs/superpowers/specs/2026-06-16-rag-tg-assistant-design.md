# Дизайн: RAG-ассистент в Telegram (флагман-портфолио)

Дата: 2026-06-16 · Автор: Роман Щербин

## Цель
Флагманский портфолио-проект, который работает на два фронта:
- **Клиентам Kwork** — живой бот, которого можно потрогать («напиши и спроси»).
- **Работодателям** — чистый GitHub-репозиторий, показывающий глубину: RAG + guardrails + eval.

## Демо-бизнес
Вымышленный интернет-магазин товаров для дома **«ДомОк»** (название можно сменить). База знаний ~15–30 разделов: каталог/категории, прайс, доставка, оплата, возврат, гарантия, FAQ. Контент базы — на русском (целевой рынок РФ).

## Что делает бот
Пользователь пишет вопрос → бот отвечает **строго по базе знаний**, с указанием источника. Если ответа в базе нет — честно сообщает «не знаю / уточните у менеджера», НЕ выдумывает.

## Компоненты (изолированные, тестируемые)
1. **Ingestion** — загрузка документов → чанкинг → эмбеддинги в векторную БД.
2. **Retrieval** — поиск релевантных фрагментов по вопросу.
3. **Answer-агент** — Claude формирует ответ только по найденным фрагментам + цитирование источника; guardrail против галлюцинаций.
4. **Telegram-бот** — интерфейс (/start, вопрос-ответ).
5. **Eval-харнесс** ⭐ — набор тест-вопросов + автопроверка качества (релевантность ответа, отсутствие выдумок). Ключевой отличитель от no-code-решений.
6. **Логи / observability** — что спрашивали, что отвечал, какие источники использованы.

## Глубина (для работодателей)
- Anti-hallucination guardrail (отказ при отсутствии ответа в базе).
- Цитирование источников.
- Eval-pipeline (тест-сет + автооценка; при наличии — LLM-as-judge).
- Логирование запросов и ответов.

## Стек
Python · Claude API · векторная БД (Chroma или FAISS) · Telegram Bot API · деплой (Modal/Render/VPS) · GitHub. README репозитория — на английском (в стиле других репо Романа), контент базы — на русском.

## Артефакты (3)
1. **Живой бот** в Telegram (публичный, любой может написать).
2. **GitHub-репозиторий** с README: архитектура, запуск, результаты eval.
3. **Мини-кейс** (1 страница / в README): задача → решение → метрики качества.

## Scope MVP (YAGNI — НЕ делаем)
Админ-панель, мультиканал (сайт/WhatsApp), платежи, мультиязык, дообучение. Всё это — опции в кворках, для флагмана не нужны.

## Критерии успеха
- Бот корректно отвечает по базе и честно отказывается вне базы (проверяется eval-сетом).
- Репозиторий запускается по README с нуля.
- Eval показывает измеримое качество (доля корректных ответов, ноль галлюцинаций на тест-сете).
